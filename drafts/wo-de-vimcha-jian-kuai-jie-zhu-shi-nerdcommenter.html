我的vim插件--快捷注释NerdCommenter

2014年07月28日

NERDCommenter简介

NERDCommenter是我用过最好用的注释插件，支持几乎所有的文件类型，如果没有什么特别的需求，安装上以后就可以直接使用了，不需要做任何配置。

NERDCommenter安装

NERDCommenter使用

  • [n]<leader>cc 注释当前行(普通模式)或者选中的字符(可视模式-块)
  • [n]<leader>cn 同[n]<leader>cc,但是如果已被注释，强制执行嵌套
  • [n]<leader>c<space> 注释/取消注释选择的行,注释还是取消注释取决于选择的第一行
  • [n]<leader>cm 用最少的注释符注释,如用/**/注释多行c语言，而不是每一行都用/**/
  • [n]<leader>ci 反转注释状态，对每行分别执行，没注释的注释，已注释的取消
  • [n]<leader>ca 切换注释方式，如js可以在//和/**/两种方式间切换使用
  • [n]<leader>cs 用性感注释(sexy comment)对所选行进行注释
  • [n]<leader>cu 取消注释
  • [n]<leader>cy 同[n]<leader>cc，注释前先复制了当前行
  • [n]<leader>cl 注释并将注释符对齐（左侧注释符对齐）
  • [n]<leader>cb 注释并将注释符对齐(两端的注释符都对齐)
  • <leader>c$ 对从光标到行末的字符进行注释
  • <leader>cA 在本行末增加注释

所谓性感注释如下所示

/*
 * this is sexy comment
 * this is sexy comment
 * this is sexy comment
 */

作者把注释的行为分得非常细,我个人用的比较多的是<leader>c<space>,<leader>cA,

NERDCommenter的配置

插件开关，默认1开，0关

let loaded_nerd_comments=1

自定义键盘映射,一般很少会更改这些映射，除非和其他插件冲突,但是这些英文变量的名称有助于理解和记忆命令本身。NERDCreateDefaultMappings是否定义默认映射，默认为1

let NERDCreateDefaultMappings = 1
nmap <plug>NERDCommenterComment <leader>cc
nmap <plug>NERDCommenterNest <leader>cn
nmap <plug>NERDCommenterToggle <leader>c<space>
nmap <plug>NERDCommenterMinimal <leader>cm
nmap <plug>NERDCommenterInvert <leader>ci
nmap <plug>NERDCommenterSexy <leader>cs
nmap <plug>NERDCommenterYank <leader>cy
nmap <plug>NERDCommenterToEOL <leader>c$
nmap <plug>NERDCommenterAppend <leader>cA
nmap <plug>NERDCommenterAltDelims <leader>ca
nmap <plug>NERDCommenterAlignLeft <leader>cl
nmap <plug>NERDCommenterAlignBoth <leader>cb
nmap <plug>NERDCommenterUncomment <leader>cu

如果需要在插入模式下使用,需要配置键位映射, TODO注:配置后一直没有生效

let NERDComInsertMap='<c-c>'

是否嵌套注释，默认1是，如果设成0，则用<leader>cc时如果已注释就不再添加注释符

let NERDDefaultNesting = 1

NERDDefaultNesting 性感注释的样式:默认是0,可选值：0，1

let NERDCompactSexyComs = 1

/* 这是样式1
 * 这是样式1
 * 这是样式1*/
/*
 * 这是样式0
 * 这是样式0
 * 这是样式0
 */

还有几个配置参数如NERDAllowAnyVisualDelims、NERDBlockComIgnoreEmpty、NERDCommentWholeLinesInVMode规定了注释在可视模式(包括可视模式、可视块模式、可视行模式)下的行为细节，极少会用到，一般使用默认即可，如果想了解这些参数及另外的可配置项,可通过h NERDCommenter查看帮助文档。

存在的问题

只能根据文件类型提供注释，在html中的css或者javasciprt就无法识别了。
